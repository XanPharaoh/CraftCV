     TextButton(
         onClick = {
             val intent = Intent(Intent.ACTION_SENDTO).apply {
                 data = Uri.parse("mailto:feedback@craftcv.app")
                 putExtra(Intent.EXTRA_SUBJECT, "CraftCV Feedback")

     }

  <action>
  Audit all screens and components for accessibility. Focus on these areas:

  **1. Content descriptions for Icons:**
  Search all `Icon(` calls. Every `Icon` that conveys meaning must have a non-null `contentDescription`. Icons that are purely decorative (next to text that already describes the action) can use `contentDescription = null`.

  Common fixes:
  - Navigation icons: "Go back", "Open history", "Open settings"
  - Action icons: "Upload resume", "Submit", "Download"
  - Status icons: "Pro badge", "Uses remaining"
  - ScoreGauge: Add `Modifier.semantics { contentDescription = "ATS Score: $score percent" }`

  **2. Touch target sizes:**
  Ensure all clickable elements are at least 48dp × 48dp. Material3 IconButton already defaults to 48dp minimum. Check for:
  - Custom clickable Text without padding
  - Small icons used as buttons outside IconButton wrapper
  - If any are found, wrap in `Modifier.sizeIn(minWidth = 48.dp, minHeight = 48.dp)`

  **3. ScoreGauge accessibility:**
  The Canvas-based ScoreGauge currently has no text alternative. Add semantics:
  ```kotlin
  Canvas(
      modifier = Modifier.fillMaxSize().padding(innerPadding)
          .semantics { contentDescription = "ATS compatibility score: ${score} percent" }
  ) { ... }
  ```

  **4. Form field labels:**
  Ensure TextField/OutlinedTextField on ProfileScreen and DashboardScreen have proper `label` or `placeholder` parameters (they likely already do, but verify).

  This is an audit-and-fix task. Read each file, identify missing accessibility attributes, and add them. Don't restructure layouts — only add `contentDescription`, `semantics`, and touch target fixes.
